# v2.0.4

Release date: 2026-03-12

## Highlights

- **File Attachments** тАФ Upload files to tasks and projects (50MB per file, 5 per request). Text files auto-inline into agent execution prompts; binary files listed with paths.
- **Full-Stack Docker Deployment** тАФ Production `docker-compose.deploy.yml` with three containers: app, Qdrant (vector search), Ollama (local embeddings). Bind-mount volumes for persistence.
- **Workflow Pack Management** тАФ Create custom packs, preview deletion impact, delete with agent reassign/delete options.
- **Login Gate** тАФ Optional browser-side authentication gate before accessing the dashboard.
- **SQLite Auto-Backup** тАФ POST `/api/backup` creates timestamped backup; auto-backup on startup (retains 5 most recent).
- **Vector Search Integration** тАФ Qdrant + Ollama nomic-embed-text for semantic search across tasks and meetings.

## New API Endpoints

| Method | Path | Purpose |
| ------ | ---- | ------- |
| POST | `/api/attachments/:ownerType/:ownerId` | Upload files (multipart) |
| GET | `/api/attachments/:ownerType/:ownerId` | List attachments |
| GET | `/api/attachments/download/:id` | Download file |
| DELETE | `/api/attachments/:id` | Delete attachment |
| POST | `/api/workflow-packs` | Create custom workflow pack |
| GET | `/api/workflow-packs/:key/impact` | Preview pack deletion impact |
| DELETE | `/api/workflow-packs/:key` | Delete workflow pack |
| POST | `/api/backup` | Create SQLite backup |
| GET | `/api/vector/status` | Vector search health |
| GET | `/api/vector/search` | Semantic search |
| POST | `/api/vector/reindex` | Re-index into Qdrant |

## Schema Changes

- New `attachments` table with polymorphic `owner_type`/`owner_id` pattern
- Index: `idx_attachments_owner` on `(owner_type, owner_id, created_at DESC)`

## Frontend Changes

- Terminal panel gains "Files" tab with drag-and-drop upload zone, file list with mime icons, download/delete actions
- 4-language i18n support (ko/en/ja/zh) for attachment UI

## Docker Compose

Three compose variants available:
- `docker-compose.deploy.yml` тАФ Production with bind-mount volumes (recommended)
- `docker-compose.minimal.yml` тАФ App only, no vector search
- `docker-compose.yml` тАФ Full stack with named volumes

Node.js tuning: `--max-old-space-size=4096`, `UV_THREADPOOL_SIZE=16`, container memory limits (8G app, 2G Qdrant, 4G Ollama).

## Environment Variables

New/updated:
- `ATTACHMENTS_DIR` тАФ File storage path (default: `./attachments`)
- `QDRANT_URL` тАФ Qdrant endpoint (default: `http://localhost:6333`)
- `QDRANT_ENABLED` тАФ Enable vector search (`true`/`false`)
- `EMBEDDING_BASE_URL` тАФ OpenAI-compatible embedding endpoint
- `EMBEDDING_MODEL` тАФ Embedding model name (default: `nomic-embed-text`)

## Security

- Attachment uploads use UUID filenames to prevent path traversal
- File size enforced server-side via busboy stream limits
- Attachment cleanup on task/project deletion (files + DB rows)
